# AI 助手

TimeWrite 内置 AI 写作助手，支持流式对话和 RAG 上下文检索。

## 支持的服务商

| 服务商 | 模型 | 说明 |
|--------|------|------|
| 智谱 BigModel | `glm-4.6v` | 默认服务商，国产大模型 |
| 自定义 OpenAI 兼容 | 任意 | 支持任何 OpenAI 兼容 API |

## 配置 AI 服务

在**设置 → AI 配置**页面完成以下配置：

### 基本配置
1. **服务商**：选择智谱 BigModel 或自定义
2. **API 地址**：API 端点 URL（选择服务商自动填充默认值）
3. **对话模型**：模型名称
4. **Embedding 模型**：向量化模型名称

### 高级参数
| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| Temperature | 0-1.0 | 0.7 | 控制创意度，越高越天马行空 |
| 最大输出 Token | 1-131072 | 65536 | 控制回复长度 |

### API Key
非智谱服务商需填写 API Key，以密码形式存储。

### 连接测试
配置完成后点击「测试连接」按钮：
- 🟢 **绿色**：连接成功，显示可用模型列表
- 🔴 **红色**：连接失败，显示错误详情
- 🔵 **蓝色动画**：检测中

## 使用 AI 对话

### 打开面板
点击编辑器工具栏的 🤖 按钮，在右侧打开 384px 宽的 AI 对话面板。

### 快捷提示词
消息为空时，点击快捷提示按钮：
- 帮我续写下一段
- 优化这段对话
- 推演剧情走向
- 分析人物性格

### 流式对话
- 输入消息后按回车或点击发送
- AI 回复**实时流式显示**，逐字呈现
- 支持 Markdown 渲染（表格、代码块、列表等）
- 对话历史保留在面板中

### RAG 上下文检索
AI 对话会自动检索当前章节的相关内容作为上下文：
1. 发送消息时，系统自动从当前章节搜索相关内容
2. 检索结果作为系统提示词的背景信息
3. AI 回复时参考你的实际写作内容

> 当前使用 SQL LIKE 降级实现关键词搜索，Phase 4 计划接入 `sqlite-vec` 向量语义检索。

## 对话状态

| 状态 | 图标 | 说明 |
|------|------|------|
| 空闲 | ⭕ 灰色圆点 | 未进行连接测试 |
| 检测中 | 🔵 旋转动画 | 正在测试连接 |
| 已连接 | 🟢 对勾 | 连接正常 |
| 错误 | 🔴 警告 | 连接失败 |

## 技术说明

- 流式请求在 Rust 端处理，避免浏览器 CORS 问题
- 使用 Tauri 事件系统推送增量文本
- 非 Ollama 服务商需要 API Key 认证
- 配置通过 localStorage 持久化保存
