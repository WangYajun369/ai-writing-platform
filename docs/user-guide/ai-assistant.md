# AI 助手

TimeWrite 内置 AI 写作助手，支持流式对话和 RAG 上下文检索。

## 支持的服务商

| 服务商 | 模型 | 说明 |
|--------|------|------|
| 智谱 BigModel | `glm-5.1` | 默认服务商，国产大模型 |
| DeepSeek | `deepseek-chat` 等 | 支持推理思考模式（Thinking） |
| OpenAI | `gpt-4o` 等 | 需要 API Key |
| Ollama | `qwen2.5:7b` 等 | 本地部署，无需 API Key |
| 自定义 OpenAI 兼容 | 任意 | 支持任何 OpenAI 兼容 API |

## 配置 AI 服务

在**设置 → AI 配置**页面完成配置。v0.4.0 起 AI 配置拆分为两大独立模块：

### 对话配置（Chat Config）
1. **服务商**：选择智谱 BigModel / DeepSeek / Ollama / OpenAI 兼容
2. **API 地址**：API 端点 URL（选择服务商自动填充默认值）
3. **对话模型**：模型名称（如 `glm-5.1`、`deepseek-chat`）
4. **API Key**：各服务商独立管理（密码形式存储）
5. **思考模式**：DeepSeek/智谱推理模型可开启 Thinking，展示推理过程
6. **高级参数**：Temperature（0-1.0）、最大输出 Token（默认 131072）

### RAG / Embedding 配置（独立模块）
1. **Embedding 模型**：向量化模型名称（如 `embedding-3`）
2. **API Key**：独立于对话的 API Key（当前仅支持智谱 BigModel）
3. **生成索引**：一键触发对当前作品所有章节和世界观卡片的 Embedding 向量生成
4. **索引状态**：查看已索引/总计章节和世界观卡片数量
5. **连通性测试**：独立测试 RAG 服务的连接状态

### API Key
各服务商对话和 RAG 独立管理 API Key，以密码形式安全存储。

### 连接测试
配置完成后点击「测试连接」按钮：
- 🟢 **绿色**：连接成功，显示可用模型列表
- 🔴 **红色**：连接失败，显示错误详情
- 🔵 **蓝色动画**：检测中

## 使用 AI 对话

### 打开面板
点击编辑器工具栏的 🤖 按钮，在右侧打开 384px 宽的 AI 对话面板。

### 快捷提示词
消息为空时，点击快捷提示按钮：
- 帮我续写下一段
- 优化这段对话
- 推演剧情走向
- 分析人物性格

### 流式对话
- 输入消息后按回车或点击发送
- AI 回复**实时流式显示**，逐字呈现
- 支持 Markdown 渲染（表格、代码块、列表等）
- 对话历史按作品持久化保留

### 推理思考过程
使用 DeepSeek 或智谱推理模型时，AI 在正式回答前会先展示思考过程：
- **思考阶段**（thinking）：以可折叠区域展示推理过程
- **回答阶段**（answering）：流式输出正式回答
- **完成阶段**（done）：显示 Token/字数用量统计
- 点击展开/折叠可查看或隐藏完整的推理过程

### RAG 上下文检索
AI 对话会自动检索当前作品的相关内容作为上下文：
1. 发送消息时，系统自动搜索当前作品的相关章节和世界观卡片
2. 检索结果作为系统提示词的背景信息
3. AI 回复时参考你的实际写作内容

> **检索策略**：向量检索优先（有 Embedding 索引时使用余弦相似度），否则降级为 SQL LIKE 关键词搜索。

### 对话管理
- 对话按作品独立存储，切换作品时自动加载对应对话历史
- 支持清空当前作品所有对话记录
- 对话记录持久化到 localStorage，重启应用后恢复

## 对话状态

| 状态 | 图标 | 说明 |
|------|------|------|
| 空闲 | ⭕ 灰色圆点 | 未进行连接测试 |
| 检测中 | 🔵 旋转动画 | 正在测试连接 |
| 已连接 | 🟢 对勾 | 连接正常 |
| 错误 | 🔴 警告 | 连接失败 |

## 技术说明

- 流式请求在 Rust 端通过 `reqwest` 处理，避免浏览器 CORS 问题
- 使用 Tauri 事件系统推送增量文本（`ai-stream-chunk` 事件）
- 支持 SSE（OpenAI 兼容）和 NDJSON（Ollama）两种流式协议
- 非 Ollama 服务商需要 API Key 认证
- AI 配置通过 localStorage 持久化保存，自动兼容旧版格式迁移
